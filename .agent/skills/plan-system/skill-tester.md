# 沙盒測試執行邏輯 (Skill Tester - Sandbox)

當使用者發起「測試 [Skill 名稱]」或「進入測試模式」時，請執行以下流程：

## 第一階段：環境初始化 (Setup)
1. **建立沙盒**：確保 `sandbox/` 目錄及其子目錄 (`data/`, `config/`, `reports/`) 已建立。
2. **數據同步**（可選）：詢問使用者是否需要從正式資料夾複製現有數據作為測試樣本。
   - 若需要，執行：`cp -r data/* sandbox/data/` 且同步 `config/`。

## 第二階段：模擬執行 (Execution)
1. **路徑重定向**：在執行該 Skill 的過程中，**所有** 文件讀寫操作必須強制導向 `sandbox/`。
   - 正式：`/data/daily/` -> 測試：`sandbox/data/daily/`
   - 正式：`/reports/` -> 測試：`sandbox/reports/`

2. **模式選擇**：
    - **A. 自動生成測試 (Automation Test)**：AI 根據假設數據快速生成全鏈條檔案（即剛才執行的模式）。
    - **B. 完整互動測試 (Live Interactive Test)**：AI 真的啟動詢問流程，與使用者進行一問一答的對話，最後將結果存入沙盒。

3. **執行步驟 (針對互動測試)**：
    - 宣告：「進入 [Skill 名稱] 互動測試模式」。
    - 嚴格依照該 Skill 的「互動機制」進行問答。
    - 最終驗證：將對話結果寫入沙盒檔案，並呈報結果。

## 第三階段：環境清理 (Teardown)
1. **詢問清理**：測試完成後，主動詢問「測試已結束，是否要清空沙盒環境？」
2. **執行刪除**：使用者確認後，執行 `rm -rf sandbox/*` 並重新建立空的子目錄，保持整潔。

---

## 🛡️ 測試準則
- **隔離第一**：絕對禁止在測試模式下寫入非 `sandbox/` 路徑的檔案。
- **透明度**：在測試開始前，明確告知使用者：「目前已進入沙盒測試模式，所有檔案將產出於 sandbox/ 目錄。」
- **防錯**：若使用者未指定測試模式即進行調整後的首測，應主動建議：「建議先使用 `skill-tester` 在沙盒中跑一次流程。」
