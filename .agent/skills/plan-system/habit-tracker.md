# 每日紀錄執行邏輯
當使用者發起「開始紀錄」或「紀錄今日」時，請執行以下互動流程：

1. **讀取配置**：
   - 讀取 `./config/核心習慣.md`。
   - 提取「核心習慣」下方的所有項目。

2. **逐項詢問 (Interative Mode)**：
   - **不要一次列出所有問題。**
   - 請依照清單順序，**一次只問一個習慣**。
   - 例如：「首先，今天喝水有超過 3000ml 嗎？」
   - 等使用者回答後，再問下一個。

3. **靈活判定**：
   - 若使用者中途說「後面都完美」，則剩下的項目直接標註為 ✅。
   - 每一項詢問時，請保持親切、簡短的口吻。

4. **狀態評估 (NEW)**：
    - 在詢問完習慣後，請增加一個微小步驟：「最後，今天給自己的『能量/環境情緒』打幾分呢？(1-5 分)」。
    - 取得分數後，再進行最終的「本日小結」詢問。

5. **產出日誌與小結**：
    - 最後將結果存檔至 `/data/daily/YYYYMMDD.md`。
    - **架構優化**：
      - # 📅 每日紀錄：YYYY-MM-DD
      - ## ⚡ 能量情緒指標：[1-5] / 5
      - ## 📊 習慣達成狀態
      - ## 🔗 資源與工具：[本週發現的好資源/工具]
      - ## 📝 本日心得小結

---

## 🛡️ 忠實紀錄規範
- **嚴禁猜測**：若使用者未回答某個習慣或情緒指標，應再次詢問或標註為「未記錄」，不得自行根據前幾日的表現代填。
- **原始呈現**：心得小結應忠實呈現使用者的語氣與內容，不進行過度修飾或代為總結未發生的感受。
