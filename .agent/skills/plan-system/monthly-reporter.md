# 月總結執行邏輯 (Monthly Summarizer - Lean Version)

當使用者要求「產出月總結」時，請執行以下流程：

## 第一階段：月度任務與週報彙整 (核心步驟)
1. **讀取設定**：從 `./config/月計畫.md` 讀取月目標。
2. **自動對比**：掃描該月份所有 `/log/weekly/YYYY-Wxx.md`，比對任務達成狀態。
3. **對話式問答**：暫停後續動作，針對未達成或狀態不明的月目標，進行一次性精簡確認。

## 第二階段：報表生成 (僅在問答完成後執行)
1. **計算月平均執行分**：公式為 `(該月所有週執行分之總和 / 週數)`。
2. **月度習慣總覽**：產出 W1 ~ W5 的習慣達成趨勢表，標註每週達成次數。
3. **深度生活回顧 (精簡敘事版)**：
   - **核心標語**：根據本月生活基調，提煉一個具代表性的核心標語（如：從 0 到 1 的緩步推進）。
   - **精簡敘事原則**：
     - **不修飾、不評論、不給建議**：嚴禁 AI 腔調（如：做得很好、再接再厲）。
     - **關鍵紀實**：從每週日誌中擷取「最具生活感」或「重大進度」的 3-5 個片段。
     - **敘事風格**：將流水帳轉化為具有時間感的短篇紀實，字數控制在 300-500 字內，強調真實生活細節。

## 第三階段：存檔
- 路徑：`/log/monthly/YYYY-MM.md`
- 架構：
  - # 📅 月度成長紀實：YYYY-MM
  - ## 🏷️ 月度核心標語：[自動生成]
  - ## 📊 月平均執行評分：[百分比]%
  - ## 🎯 月計畫目標達成狀況
  - ## 🗓️ 習慣達成趨勢表 (W1-W5)
  - ## 📝 深度生活回顧 (精華敘事)
