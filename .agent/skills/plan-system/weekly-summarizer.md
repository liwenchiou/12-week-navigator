# 週總結執行邏輯
當使用者要求「產出週總結」時，請嚴格執行以下流程：

## 第一階段：週計畫互動確認 (核心步驟)
1. **讀取設定**：從 `./config/週計畫.md` 讀取該週任務。
2. **自動對比**：掃描該週每日日誌，標註日誌中已提及完成的項目。
3. **對話式問答**：**請暫停後續動作**，針對日誌中未提及、或未在 config 檔中標註為 `[x]` 的任務，一次一個或分批詢問使用者。
   - *範例：*「本週計畫中有幾項我還不確定進度，想跟你確認一下：『React 專題推進』與『慢跑運動』目前完成度如何？」

## 第二階段：報表生成 (僅在問答完成後執行)
1. **計算執行分**：
   - 執行分 = (週計畫已達成項數 / 週計畫總項數) * 100。
   - 格式：`分數：XX / 100 (百分比：XX%)`。
2. **繪製習慣熱點圖**：
   - 遍歷 `/log/daily/` 中該週 7 天檔案。
   - 產出表格，橫軸為一至日，縱軸為 `./config/核心習慣.md` 的項目，達成標註 ✅，未達成標註 ❌。
3. **深度生活回顧**：
   - 匯集 7 天的所有感性敘述，請**保留細節並進行文學化、敘事化的整理**（例如：將流水帳轉化為具有情感厚度的生活週記）。

## 第三階段：存檔
- **路徑**：`/log/weekly/YYYY-Wxx.md`
- **架構**：
  - # 📊 12週計畫週報：YYYY-Wxx
  - ## 🏆 本週執行評分：[分數/100] ([百分比]%)
  - ## 🎯 週計畫任務狀態
  - ## 📈 習慣養成熱點圖
  - ## 📖 深度生活回顧 (精華版)
