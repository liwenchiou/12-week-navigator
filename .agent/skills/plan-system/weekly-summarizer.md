# 週總結執行邏輯
當使用者要求「產出週總結」時，請嚴格執行以下流程：

## 第一階段：週計畫互動確認 (核心步驟)
1. **讀取設定**：從 `./config/週計畫.md` 讀取該週任務。
2. **自動對比**：掃描該週每日日誌，標註日誌中已提及完成的項目。
3. **對話式問答**：**請暫停後續動作**，針對日誌中未提及、或未在 config 檔中標註為 `[x]` 的任務，一次一個或分批詢問使用者。
   - *範例：*「本週計畫中有幾項我還不確定進度，想跟你確認一下：『React 專題推進』與『慢跑運動』目前完成度如何？」

## 第二階段：報表生成 (僅在問答完成後執行)
1. **計算執行分**：
   - 執行分 = (週計畫已達成項數 / 週計畫總項數) * 100。
   - 格式：`分數：XX / 100 (百分比：XX%)`。
2. **繪製習慣熱點圖**：
   - 遍歷 `/data/daily/` 中該週 7 天檔案。
   - 產出表格，橫軸為一至日，縱軸為 `./config/核心習慣.md` 的項目，達成標註 ✅，未達成標註 ❌。
3.  **深度生活回顧 (分類詳述版)**：
    - 彙整 7 天的所有感性敘述，**嚴禁過度精簡**。
    - **分類描述原則**：將碎片的日誌內容依據性質分類（如：技術成長、職場體悟、家庭生活、個人管理等）。
    - **保留細節**：必須保留具體的人物、地點、事件與當下的情感厚度。
4.  **數據提煉 (月報預備項)**：
    - **🌟 本週核心金句**：**必須詢問使用者**本週是否有想紀錄的一句話，或是最有力量的感悟。嚴禁自動抓取。
    - **💎 本週關鍵獲益**：挑選 1-3 個本週最具代表性的實質進步。
    - **🚧 阻礙與調整**：針對本週「未達成」的項目，**必須逐項詢問使用者未完成的原因**。嚴禁代為猜測原因，也不需要產出「下週對策」。
    - **🔗 資源與工具**：僅保留此標題區塊，由使用者後續自行填寫。嚴禁自動彙整日誌內容，也不需要詢問。

## 第三階段：存檔
- **路徑**：`/data/weekly/YYYY-Wxx.md`
- **架構**：
  - # 📊 12週計畫週報：YYYY-Wxx
  - ## 🏆 本週執行評分：[分數/100] ([百分比]%)
  - ## 🎯 週計畫任務狀態
  - ## 📈 習慣養成熱點圖
  - ## 📖 深度生活回顧
  - ## 🚧 阻礙與調整
  - ## 🌟 本週核心金句
  - ## 💎 本週關鍵獲益
  - ## 🔗 資源與工具

## 🛡️ 忠實紀錄規範
- **拒絕腦補**：針對週計畫中未完成的項目，僅能記錄使用者回覆的文字原因。若使用者僅說「沒做」，則記錄「沒做」，不得自行解讀為「體能不足」或「環境干擾」。
- **資源隔離**：資源與工具區塊必須維持使用者手動填寫的純度，不得抓取日誌中的瑣碎提及。
