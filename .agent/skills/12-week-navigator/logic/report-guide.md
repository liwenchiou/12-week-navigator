
# 📊 報表產出與總結導覽 (Report Guide)

當你需要進行週回顧、月總結或產出視覺化網頁時，請執行以下導引：

### 1. 產出本週週報 (weekly-summarizer)
- **怎麼開始**：輸入「**產出本週週報**」或「**週報**」。
- **內容**：
  - 1. **🛡️ 分層校驗 (JIT Check)**：**僅掃描當週 7 天日誌**。確認標題 Schema 是否正確。
  - 2. **執行評分**：計算本週計畫達成率與分數。
  - 3. **阻礙分析**：一問一答確認失敗原因。
  - 4. **進化對策**：為下週設定修正動作。

### 2. 產出月度報告 (monthly-reporter)
- **怎麼開始**：輸入「**執行月度總結**」或「**月報**」。
- **內容**：
  - 1. **🛡️ 分層校驗 (JIT Check)**：**僅檢查當月 4-5 份週報** 實體檔案是否存在且格式正確。
  - 2. **數據彙整**：自動擷取本月所有週報的關鍵獲益與評分。
  - 3. **戰略比對**：逐條確認月計畫達成狀況。
  - 4. **二字標語**：提供本月進化標語提案。

### 3. 生成視覺化網頁 (monthly-visualizer)
- **怎麼開始**：輸入「**產出網頁**」或「**視覺化**」。
- **內容**：
  - 1. **🛡️ 分層校驗 (JIT Check)**：確認**當月月報 MD 檔案**是否存在。
  - 2. **風格預選**：選擇條列式或段落式內容。
  - 3. **設計提案**：調用 UI/UX 腳本，展示配色方案。
  - 4. **封裝 HTML**：產出具備 React/Tailwind 質感的互動報表。

---
**💡 提示**：建議每週日晚上執行週報，每月 1 號執行月報，確保 PDCA 循環不中斷。
