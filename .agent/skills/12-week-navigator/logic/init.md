
# 🛠️ 系統初始化與預檢 (System Init)

執行此工作流以確保開發環境、資料夾結構與配置檔案格式完整且正確。

### 1. 🔍 開發環境檢查 (Environment Check)
- **GitHub CLI 檢查**：
  - 執行 `gh --version`。若未安裝，提示：「未偵測到 GitHub CLI，自動化 PR 功能將無法運作。建議執行 `brew install gh` 進行安裝。」
- **Python 3 環境檢查**：
  - 執行 `python3 --version`。Python 3 是執行 `ui-ux-pro-max` 視覺化腳本的必要環境。

### 2. 📁 資料夾與配置檔檢查 (Structure & Setup)
- **掃描目錄**：檢查 `workspace/config/`, `workspace/data/`, `workspace/reports/` 是否存在。若缺失，應主動建立。
- **配置檔完整性**：
  - 確認 `週計畫.md`, `核心習慣.md`, `月計畫.md` 實體檔案是否存在於 `workspace/config/`。
  - **🚨 缺失處理**：若偵測到缺失，逐一詢問使用者：「偵測到 `[檔案名稱]` 缺失，是否要從 **`resources/samples/`** 複製標準範本進行初始化？」

### 3. ⚙️ 配置檔格式校準 (Config Lite-Lint)
確認當前執行所需之核心配置格式正確：
- **核心習慣**：確認具備 `## 核心習慣清單`。
- **週/月計畫**：確認具備對應當前日期的區域標題。

### 4. 🔗 階層化校驗預告 (JIT Validation)
本系統採用「即時觸發校驗 (Just-In-Time)」策略以節省資源：
- **週報時**：僅掃描當週 7 天日誌。
- **月報時**：僅掃描當月 4-5 份週報。
- **網頁渲染時**：僅確認當月月報 MD 檔案。

### 5. 📝 報告與修復 (Reports & Fix)
- **回報狀態**：條列所有檢查項目的狀態 (✅/❌)，包含配置檔、歷史數據 Schema 與日期連續性。
- **互動修復**：針對偵測到的格式錯誤，先列出「預計修復後的內容」，並詢問：「是否要自動修正這些格式錯誤？」

---
**💡 提示**：完成此步驟後，系統環境即達到最佳狀態，可以放心啟動 **`/12-week-navigator`** 開始紀錄。
