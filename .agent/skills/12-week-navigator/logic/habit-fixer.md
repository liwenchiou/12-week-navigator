
# ⏳ 補記與修正精靈 (Habit Fixer)

當使用者要求「補記」、「修正日誌」或「改錯」時，請啟動此引導工作流：

1. **偵測漏缺 (Detection)**：
   - 掃描各週計畫 `YYYY-WW.md` 對應的日誌日期。
   - 向使用者回報：「目前偵測到 Wxx 有以下日期尚未紀錄：[日期1], [日期2]...」。

2. **選擇模式**：
   - 使用 **「逐項互動原則」** 詢問：「您想要 A. 補記缺失日期 還是 B. 修正已存在的日誌內容？」

3. **執行模式 A：補記缺失**：
   - **分段問答**：確定日期後，調用 `habit-tracker` 技能，但將存檔路徑改為該特定日期（例如：`20260301.md`）。
   - 嚴禁合併問題，按習慣清單逐一詢問。

4. **執行模式 B：內容修正**：
   - **讀取分析**：讀取目標日期的 `.md` 檔案，向使用者展示目前的各項數據。
   - **精確修改 (一問一答)**：
     - 問：「請問哪個項目需要修改？（例如：喝水標註為達成、能量改為 5 分、或修改心得）」
     - 取得回答後，單次執行 `replace_file_content` 更新對應區塊。
   - 詢問是否還有其他需要修改的項目，直到完成。

5. **存檔與驗證**：
   - 完成後確認實體檔案已更新。
   - 詢問是否需要重新同步週計畫狀態？（若有變動）。
