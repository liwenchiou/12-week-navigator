# 系統啟動工作流 (System Warm-up)

當使用者在全新環境啟動，或是首次在此專案呼叫 Agent 時，請執行以下步驟：

1. **讀取靈魂文件**：主動載入並分析 `.agent/instructions.md`。
2. **裝載技能組**：掃描並理解 `.agent/skills/` 下的所有 .md 檔案。
3. **環境檢查 (Prerequisite Check)**：
   - 檢查 GitHub CLI (`gh`) 是否已安裝：執行 `gh --version`。
     - 若未安裝，主動告知使用者：「未偵測到 GitHub CLI，自動化 PR 功能將無法運作。建議執行 `brew install gh` 進行安裝。」
   - 檢查 Python 3 是否已安裝：執行 `python3 --version`。
     - Python 3 是運作 `ui-ux-pro-max` 搜索腳本的必要環境。若未安裝，請告知使用者安裝。
   - 檢查 `data/` 目錄是否存在，若無則依結構建立。
   - 檢查 `.gitignore` 是否已正確配置。
4. **現狀彙整**：
   - 掃描 `config/`，跟使用者簡報目前的目標（月目標、核心習慣）。
   - 掃描 `data/daily/` 最新的紀錄日期，告知使用者系統目前所在的進度位置。
5. **就緒宣告**：
   - 回報「系統已準備就緒，隨時可以開始導航」。
   - **🚀 建議開始的步驟**：
     - 提供具體的互動選項，並包含「怎麼開始」的關鍵字。
     - 例如：
       - **執行今日紀錄 (habit-tracker)**：
         - **怎麼開始**：輸入「紀錄今天」或「開始紀錄」。
         - **內容**：說明紀錄的具體行為。
       - **執行測試 (skill-tester)**：
         - **怎麼開始**：輸入「執行測試」或「測試 [Skill 名稱]」。
         - **內容**：說明沙盒測試的保護機制。
       - **回顧近期進度**。
       - **自定義任務**。
